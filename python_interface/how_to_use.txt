Програма потребує 2 типи файли на вхід: “.conf”, “txt”. Перший з них має містити параметри заданого фізичного об’єкту, а другий його матрицю чисельної моделі. Для автоматичної генерації моделі потрібно запустити інтерфейс написаний на мові програмування python 3.10 за відносним шляхом “\python interface\”. При його запуску викликається візуальний інтерфейс в якому знаходиться поля для заповнення та кнопка збереження всіх параметрів. Вони  є обов’язковими, тому якщо хоча б одне з них буде порожнім, то про таку помилку буде повідомлено. Всі поля параметрів об’єкту мають бути раціонального типу, про недотримання цієї умови буде повідомлятися так само через інтерфейс. Також у верхній частині інтерфейсу знаходяться поля в яких потрібно вказати шлях до файлів у яких будуть збережені дані вже вказані користувачем або такі файли будуть створенні програмою. При відсутності або некоректності вказаного шляху буде виведено попередження про це.
Далі знаходяться 3 позначених поля “x”,”y”,”temperature”, які потрібні для формування матриці температурної чисельної моделі об’єкту. За вище вказаними параметрами “coordinate size x” та  “coordinate size y” буде створена початкова матриця заповнена числом що вказане в  полі “standard temperature”. Розмір у ширину “coordinate size x”, а у висоту “coordinate size y”. “x” використовується для задання значень які будуть використовуватися  в русі за системою координат, також можна вказати значення “x”, що буде означати, що значення буде відповідне X-овій координаті. У наступному полі “y” потрібно вказати просту функцію, що може включати віднімання, додавання, ділення,  множення та піднесення до степені. Також на самому початку можна вказати знак нерівності, що відповідно перетворить функцію на нерівність. В останньому полі потрібно вказати температурне значення, яке буде розташовуватися в системі координат за параметрам, що були вказані в 2 попередніх полях. Кількість таких функцій може бути майже необмеженою. Для того, щоб створити наступну умову, результат якої буде накладатися поверх й заміняти попередні значення на відповідний до себе, потрібно її написати в новому рядку, але кількість таких рядків має бути однаковою, у разі недотримання цього правила інтерфейсом буде повідомлено таку помилку. Останньою можливістю генератора є ячейка “rotate on 90°”, вибравши яку остаточну матрицю буде повернуто за часовою стрілкою на 90°, це потрібно для можливості зміни залежності змінних системи координат.


The program requires 2 types of input files: ".conf", "txt". The first has the parameters of a given physical object, and the other its matrix of a numerical model. For the automatic generation model, you need to run the interface written in python 3.10 language programming using "\ python interface \". When you start it, there is a visual interface button that has fields to add and save all the parameters. They are required, so if at least one of them is empty, such an error will be known. All parameters of the object have the principle of rational type, for non-compliance with these conditions will be reported in the same way through the interface. Also at the top of the interface are fields in which you need to specify the path to the files in which the data already specified by the user will be saved or such files will be created by the program. If this path is missing or incorrect, a warning will be displayed.
Then there are 3 marked fields "x", "y", "temperature", which are required to form a matrix of temperature numerical model of the object. The above parameters "coordinate size x" and "coordinate size y" will create an initial matrix filled with the number specified in the field "standard temperature".
Size in width "coordinate size x", and in height "coordinate size y". "X" is used to specify the values ​​that will be used in the movement of the coordinate system, you can also specify the value of "x", which will mean that the value will correspond to the X coordinate. In the next field "y" you need to specify a simple function that can include subtraction, addition, division, multiplication and exponentiation. You can also specify an inequality sign at the beginning, which will turn the function into an inequality. In the last field you need to specify the temperature value, which will be located in the coordinate system according to the parameters specified in the 2 previous fields. The number of such functions can be almost unlimited.
In order to create the next condition, the result of which will be on top and replace the previous values ​​with the corresponding one, you need to write it in a new line, but the number of such lines will be the same, if you do not follow this interface, you will know this error. The last power of the generator is the cell "rotate on 90 °", selecting a final matrix will be rotated clockwise by 90 °, this is necessary to be able to change the variable coordinate systems.
